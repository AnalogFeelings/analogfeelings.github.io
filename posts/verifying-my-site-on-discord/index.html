Verifying my website on Discord

Posted on 2023-09-11 17:43 Never Edited

Recently, Discord added a new feature for some users that allows them to connect their website to their account.
This allows you to show off your website next to your other social media accounts, which is pretty neat.

I was one of the lucky ones who got the feature early, because if you didn't know, Discord rolls features out gradually instead of immediately.
So off I went trying to verify my website!

The Required Steps

First, Discord will ask you to enter the domain name of your website into a text field, and click Next.

Important The domain name must not contain the https:// prefix or anything else.
For example, google.com is valid, but https://google.com/ is not.

Then, it will show you two verification options, DNS record or HTTP.
I chose HTTP because I do not own a real domain.

The HTTP method asks you to create a folder called .well-known in the root of the website. You must then add a file called discord (without file extension), and paste in the string the client shows you.
This way, you should be able to verify your domain correctly, in theory.

The Bullshit Begins

So, it turns out that if you don't have a real website with a server and a domain, Discord will not be able to find the verification file.
The reason for this is because of Jekyll, the static site generator that GitHub Pages uses by default.

Jekyll ignores directories which begin by a period, which means the built site will not contain said directories.
This seems to be related to the UNIX philosophy of marking such directories as "hidden".

Luckily, this behavior can be modified to fit our needs.
